# Contributing to PAI for Windows

Thanks for your interest in contributing.

## Getting Started

1. Fork the repo and clone locally
2. Follow the [Quick Start](README.md#quick-start) to get PAI running
3. Create a feature branch: `git checkout -b feat/your-feature`
4. Make your changes and test them
5. Open a pull request with a clear description

## What We Welcome

- **New skills** — Add `.md` workflow files to `skills/`
- **Setup improvements** — Better scripts, edge case handling
- **Documentation** — Clearer guides, troubleshooting tips
- **Bug reports** — Open an issue with steps to reproduce

## Code Style

- Shell scripts: POSIX-compatible where possible, Bash when needed
- TypeScript: Bun runtime, strict mode
- Markdown: CommonMark

## Issues

Search existing issues before opening a new one. Include:
- OS and WSL version
- Bun version (`bun --version`)
- Steps to reproduce
- Expected vs actual behavior
